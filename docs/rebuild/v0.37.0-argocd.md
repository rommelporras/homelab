# v0.37.0 - ArgoCD Installation & Bootstrap

> **Release:** v0.37.0
> **Phase:** 5.7 (ArgoCD Installation & Bootstrap)
> **Goal:** Deploy ArgoCD as the GitOps engine for declarative cluster management
> **Prerequisite:** [v0.29.0-vault-eso.md](v0.29.0-vault-eso.md) completed, Vault unsealed, ESO running

---

## Overview

```
GitHub (public repo — source of truth for manifests and Helm values)
      |
ArgoCD (argocd namespace — 6 pods, non-HA)
      |
      ├── application-controller — reconciles Applications against cluster state
      ├── server — UI + API at https://argocd.k8s.rommelporras.com
      ├── repo-server — clones repos, renders Helm templates
      ├── applicationset-controller — manages ApplicationSets
      ├── notifications-controller — Discord webhook alerts
      └── redis — ephemeral cache (no PVC)
```

**Key design choices:**
- Non-HA (single replica per component) — saves ~1.5 CPU and ~2.3Gi memory vs HA
- ArgoCD downtime only blocks new syncs — running applications are unaffected
- Public GitHub repo — no deploy token or credentials needed for repo access
- Self-management Application — ArgoCD's own config tracked in Git
- Redis is ephemeral — no backup needed, cache rebuilds on restart
- Dex SSO disabled — local admin account only

## Step 2: Seed Vault with ArgoCD Secrets

Run in trusted terminal with `op` and `vault` access:

```bash
# Generate credentials
ADMIN_PASS=$(openssl rand -base64 24)
ADMIN_HASH=$(htpasswd -nbBC 10 "" "$ADMIN_PASS" | tr -d ':\n' | sed 's/$2y/$2a/')
SERVER_KEY=$(openssl rand -hex 16)

echo "=== Save your ArgoCD admin password ==="
echo "$ADMIN_PASS"
echo "======================================="

# Create 1Password item (store BOTH plain password and bcrypt hash)
op item create \
  --vault "Kubernetes" \
  --category "Login" \
  --title "ArgoCD" \
  "password=$ADMIN_PASS" \
  "admin-password=$ADMIN_HASH" \
  "server-secret-key=$SERVER_KEY"

# Add gitops webhook to shared Discord Webhooks item
op item edit "Discord Webhooks" --vault "Kubernetes" "gitops=<your #gitops webhook URL>"

# Seed Vault
vault kv put secret/argocd \
  admin-password="$(op read 'op://Kubernetes/ArgoCD/admin-password')" \
  discord-webhook-url="$(op read 'op://Kubernetes/Discord Webhooks/gitops')" \
  server-secret-key="$(op read 'op://Kubernetes/ArgoCD/server-secret-key')"
```

## Step 5: Apply Networking

```bash
# HTTPRoute for UI access
kubectl-admin apply -f manifests/argocd/httproute.yaml

kubectl-admin apply -f manifests/argocd/networkpolicy.yaml

# Verify UI accessible
curl -sk https://argocd.k8s.rommelporras.com | head -1
# Expected: <!doctype html>
```

**CiliumNP egress includes:**
- DNS with FQDN inspection (`rules: dns: matchPattern: "*"`)
- GitHub + `*.github.io` + `*.githubusercontent.com` (repo cloning + chart downloads)
- kube-apiserver (resource management)
- Discord (notifications)

## Gotchas

- **Redis image override** — chart default `ecr-public.aws.com` triggers VAP warning. Values override to `public.ecr.aws`.
- **CiliumNP toFQDNs** — requires DNS inspection rule (`rules: dns: matchPattern: "*"`) in same policy. Without it, FQDN-to-IP cache never populates and HTTPS connections time out.
- **HTTPRoute port** — service port is 80 (maps to container 8080). Not 8080.
- **Self-management ComparisonError** — expected until `helm/argocd/values.yaml` is pushed to GitHub.
- **ArgoCD v3.3.5 on K8s 1.35** — works despite official matrix covering 1.31-1.34. Upgrade to v3.4.0 when GA (deferred).
