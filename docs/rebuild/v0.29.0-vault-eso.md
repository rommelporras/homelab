# v0.29.0 — Vault + External Secrets Operator

> **Release:** v0.29.0
> **Phase:** 4.29 (Vault + External Secrets Operator)
> **Goal:** Replace all imperative `kubectl create secret` commands with declarative `ExternalSecret` CRDs backed by self-hosted HashiCorp Vault
> **Prerequisite:** [v0.28.0-atuin.md](v0.28.0-atuin.md) completed, all services running

---

## Overview

```
1Password (source of truth — seed once, update manually on rotation)
      │
      ▼ scripts/seed-vault-from-1password.sh (🔒 safe terminal)
      │
Vault 1-pod (vault namespace, Raft on Longhorn 5Gi)
      │
      ├── NFS NAS (daily Raft snapshots to /Kubernetes/Backups/vault, 15-day retention)
      │
ESO ClusterSecretStore — Kubernetes auth (ESO ServiceAccount → Vault validates via K8s API)
      │
ExternalSecret CRDs (committed to git, per namespace, zero hardcoded values)
      │ synced every 1h
      ▼
K8s Secrets (auto-created and kept in sync)
```

**Key design choices:**
- 1 pod (not HA) — Longhorn already replicates data 2×; unsealer recovers pod restarts in ~30s
- Auto-unsealer Deployment polls vault-0 every 30s; unseals if sealed
- `vault-unseal-keys` K8s Secret is the only remaining imperative secret (bootstrap requirement)
- VAULT_ADDR set in `~/.zshrc` — no port-forward needed for day-to-day vault CLI use

## Step 3: Initialize Vault 🔒

> Run in safe terminal (has `op` access). **Do this only once.**

```bash
export VAULT_ADDR=https://vault.k8s.rommelporras.com

# Initialize — 5 unseal keys, threshold 3
vault operator init > ~/.vault-keys && chmod 600 ~/.vault-keys

# Save to 1Password immediately (break-glass backup)
eval $(op signin)
op item create --category=login --title="Vault Unseal Keys" --vault=Kubernetes \
  "unseal-key-1=$(grep 'Unseal Key 1' ~/.vault-keys | awk '{print $NF}')" \
  "unseal-key-2=$(grep 'Unseal Key 2' ~/.vault-keys | awk '{print $NF}')" \
  "unseal-key-3=$(grep 'Unseal Key 3' ~/.vault-keys | awk '{print $NF}')" \
  "unseal-key-4=$(grep 'Unseal Key 4' ~/.vault-keys | awk '{print $NF}')" \
  "unseal-key-5=$(grep 'Unseal Key 5' ~/.vault-keys | awk '{print $NF}')" \
  "root-token=$(grep 'Initial Root Token' ~/.vault-keys | awk '{print $NF}')"

# First-time manual unseal (need 3 of 5 keys)
vault operator unseal $(grep 'Unseal Key 1' ~/.vault-keys | awk '{print $NF}')
vault operator unseal $(grep 'Unseal Key 2' ~/.vault-keys | awk '{print $NF}')
vault operator unseal $(grep 'Unseal Key 3' ~/.vault-keys | awk '{print $NF}')

# Login with root token
vault login $(grep 'Initial Root Token' ~/.vault-keys | awk '{print $NF}')

# Verify unsealed
vault status
# Expected: Sealed false, Initialized true
```

## Step 4: Create vault-unseal-keys Secret 🔒

## Gotchas

| Issue | Detail |
|-------|--------|
| `unauthenticated_metrics_access` placement | **Must be in `listener.telemetry {}` block** (not top-level `telemetry {}`). Vault 1.16+ breaking change. Top-level causes 403 on `/v1/sys/metrics`. |
| Vault StatefulSet uses OnDelete | Config changes via Helm upgrade don't restart the pod. Must `kubectl delete pod vault-0 -n vault` after `helm upgrade` to apply HCL config changes. |
| kube-prometheus-stack ignores pod annotations | Metrics scraping requires a ServiceMonitor CRD (`manifests/vault/servicemonitor.yaml`). `prometheus.io/scrape: "true"` annotations are ignored. |
| NFS directory must pre-exist | The snapshot CronJob NFS PV requires `/Kubernetes/Backups/vault` to exist on the NAS before applying. Create it first (Step 11). |
| `VAULT_ADDR` in `.zshrc` | Added to `~/.zshrc` — no manual export or port-forward needed. Source or open new terminal after first setup. |
| ESO `externalsecret.yaml` filenames | Pre-commit hook `protect-sensitive.sh` matches `*secret.yaml`. Write via bash heredoc if needed. |
| First-time unseal is manual only | After `vault operator init`, the unsealer pod won't start until `vault-unseal-keys` secret exists. Unseal manually 3 times before creating the K8s secret. |
| `upgrade-prometheus.sh` temp file overrides receivers | The temp secrets file fully replaces `alertmanager.config.receivers`. All receiver settings (including `send_resolved`) must be in the script, not just `values.yaml`. |

## Bootstrap Order After Full Cluster Loss

```
1. K8s control plane + etcd           — automatic (kubeadm)
2. Longhorn + volume attach           — automatic
3. vault-0 pod starts                 — automatic (StatefulSet)
4. vault-unsealer unseals vault-0     — automatic (~30s after pod ready)
5. ESO reconnects to ClusterSecretStore — automatic
6. All 30 ExternalSecrets re-sync     — automatic (within 1h, or on pod restart)
7. All workloads start with secrets   — automatic
```

**If Vault PVC is lost (Longhorn failure):**
1. Redeploy Vault via `helm install` (or `helm upgrade --install`)
2. 🔒 Re-initialize + manually unseal (Steps 3-4)
3. 🔒 Update 1Password "Vault Unseal Keys" + recreate `vault-unseal-keys` K8s Secret
4. 🔒 Re-run `scripts/seed-vault-from-1password.sh`
5. All ExternalSecrets re-sync automatically
