**Key design choices:**
- Per-project webhook secrets (DR-4) ã GitLab webhooks don't HMAC the payload, shared secret = cross-project forgery risk.
- Git-based image promotion (DR-1) ã CI commits `newTag` into the overlay; ArgoCD owns the deploy. Failed tests leave no commit.
- 8-char short SHA for both projects (DR-2) ã readable in `kubectl describe`, Grafana, Discord.
- SSH deploy key scoped to `rommelporras/homelab` only (DR-5) ã mounted in `deploy-image` pod only, never in build pod.
- `synchronization.mutex: deploy-image-lock` + 5-attempt rebase-retry (DR-7) ã handles both in-cluster concurrency and external writes.

## Installation Order

1. **1Password: consolidate everything under one `Argo Workflows` item** (renamed from `Argo Workflows UI`). Fields required:
   - `sso-client-id`, `sso-client-secret` (existing ã UI SSO)
   - `registry-username`, `registry-password` (new ã GitLab PAT/deploy token with `write_registry` + `read_registry` scopes)
   - `github-deploy-key` (new ã SSH private key; `ssh-keygen -t ed25519 -f id_ed25519_argo_ci -C argo-ci@k8s.rommelporras.com`; public key goes to `rommelporras/homelab` ã Settings ã Deploy keys with write access)
   - `gitlab-api-token` (new ã GitLab PAT with `api` scope; Argo Events EventSources use this to auto-register webhooks)
   - `invoicetron-webhook-secret`, `portfolio-webhook-secret`, `staging-promote-token` (new ã `openssl rand -hex 32` each; DR-4: per-project secrets to bound forgery blast radius)

   Convenience script to add the 4 Events-related fields in one shot:
   ```bash
   op item edit "Argo Workflows" --vault Kubernetes \
     "invoicetron-webhook-secret[password]=$(openssl rand -hex 32)" \
     "portfolio-webhook-secret[password]=$(openssl rand -hex 32)" \
     "staging-promote-token[password]=$(openssl rand -hex 32)"
   ```
   `gitlab-api-token` is pasted from the GitLab UI after creating the PAT (the `POST /personal_access_tokens` endpoint refuses PAT auth by design, so CLI creation is not practical).
2. **Seed Vault** - run `./scripts/vault/seed-vault-from-1password.sh` in a safe terminal (the script now includes the 7 new paths).
3. **Commit manifests** - `manifests/argo-events/*`, `helm/argo-events/values.yaml`, `manifests/argocd/apps/argo-events*.yaml`, `manifests/argocd/appprojects.yaml` (argo-events ns added to `infrastructure` project), `manifests/argo-workflows/externalsecret-*.yaml`, `manifests/argo-workflows/rbac/ci-workflow-rbac.yaml`, `manifests/argo-workflows/ciliumnetworkpolicy-ci-pipelines.yaml`, `manifests/argo-workflows/templates/{clone,lint,type-check,test-unit,test-e2e,build-image,deploy-image,verify-health,notify-on-failure,portfolio-pipeline,invoicetron-pipeline,portfolio-staging-promote}-template.yaml`, `manifests/invoicetron/overlays/{dev,prod}/networkpolicy.yaml` (added argo-workflows ingress), `manifests/kube-system/image-registry-policy.yaml` (mcr.microsoft.com allowlist), monitoring additions.
4. **ArgoCD pickup** - `argo-events` + `argo-events-manifests` Applications auto-discovered within 3 min.
5. **Verify controllers Ready** - `kubectl-homelab -n argo-events get pods,eventsources,sensors,eventbus`. EventSources auto-register webhooks in GitLab (check `Settings ã Webhooks` on each project).
6. **Smoke test** - push a trivial commit to `develop` on either project. Follow in the Argo Workflows UI at `argo-workflows.k8s.rommelporras.com`.
7. **Retire GitLab CI deploy** - once pipelines are green for 3+ days on both branches, delete the `deploy:*` jobs from both projects' `.gitlab-ci.yml` and archive the Portfolio `kube-token-*` Vault entries.

## Rollback

**Per-project:** re-enable the project's `deploy:*` GitLab CI jobs (they are `when: manual` from Stage 1, still functional), then delete that project's Sensors and EventSource.

```bash
# invoicetron rollback
kubectl-admin delete sensor -n argo-events gitlab-invoicetron-develop gitlab-invoicetron-main
kubectl-admin delete eventsource -n argo-events gitlab-invoicetron
```

**Full Stage 2 rollback:** delete the two ArgoCD Applications (argo-events, argo-events-manifests) and the `argo-events` namespace. Re-enable all GitLab CI deploy stages on both projects.
