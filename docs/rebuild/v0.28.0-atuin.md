## Step 2: Create Secrets

Create the 1Password item first, then generate the Kubernetes secret:

```bash
# In safe terminal (not Claude Code):
eval $(op signin)

# Create 1Password item (if not exists)
op item create \
  --vault "Kubernetes" \
  --category "Secure Note" \
  --title "Atuin" \
  --generate-password=32,letters,digits \
  'db-username[text]=atuin' \
  'db-database[text]=atuin' \
  'db-uri[text]=postgres://atuin:<PASSWORD>@postgres:5432/atuin'

# Read values and create K8s secret
DB_USER=$(op read 'op://Kubernetes/Atuin/db-username')
DB_PASS=$(op read 'op://Kubernetes/Atuin/db-password')
DB_NAME=$(op read 'op://Kubernetes/Atuin/db-database')
DB_URI=$(op read 'op://Kubernetes/Atuin/db-uri')

kubectl --kubeconfig ~/.kube/homelab.yaml -n atuin create secret generic atuin-secrets \
  --from-literal=POSTGRES_USER="$DB_USER" \
  --from-literal=POSTGRES_PASSWORD="$DB_PASS" \
  --from-literal=POSTGRES_DB="$DB_NAME" \
  --from-literal=ATUIN_DB_URI="$DB_URI"
```

**Important:** `db-password` must use only `[A-Za-z0-9.~_-]` — special characters like `@`, `!`, `#` break `ATUIN_DB_URI` parsing.

## Step 8: Register Accounts

**Important:** Server must have `ATUIN_OPEN_REGISTRATION=true` during this step. The manifests ship with `false` — temporarily set to `true`:

```bash
# Edit server-deployment.yaml: ATUIN_OPEN_REGISTRATION -> "true"
kubectl-homelab apply -f manifests/atuin/server-deployment.yaml
kubectl-homelab -n atuin rollout status deployment/atuin-server --timeout=60s
```

Register personal account:

```bash
atuin register -u rommel-personal -e '<email>' -p '<password>'
# Wrap password in single quotes (zsh glob expansion on * and other special chars)

# Save encryption key to 1Password
atuin key
# Copy output -> op item edit "Atuin" --vault "Kubernetes" 'encryption-key[text]=<key>'

# Import existing history
atuin import zsh
exec zsh  # Reload shell to set $ATUIN_SESSION
atuin sync
atuin status
# Expected: Username: rommel-personal, sync count > 0
```

Register EAM account:

```bash
atuin logout
atuin register -u rommel-eam -e '<eam-email>' -p '<password>'

# Verify same encryption key (by design — reuses ~/.local/share/atuin/key)
atuin key
# Should match op://Kubernetes/Atuin/encryption-key

atuin import zsh
exec zsh
atuin sync
```

Switch back to personal and close registration:

```bash
atuin logout
atuin login -u rommel-personal -p '<password>' -k "$(op read 'op://Kubernetes/Atuin/encryption-key')"
atuin sync

# Close registration: revert ATUIN_OPEN_REGISTRATION -> "false"
kubectl-homelab apply -f manifests/atuin/server-deployment.yaml
kubectl-homelab -n atuin rollout status deployment/atuin-server --timeout=60s
```

## Rollback

```bash
# Remove Atuin server (client history stays local)
kubectl-homelab delete -f manifests/atuin/
kubectl-homelab delete -f manifests/monitoring/alerts/atuin-alerts.yaml
kubectl-homelab delete -f manifests/monitoring/probes/atuin-probe.yaml
kubectl-homelab delete -f manifests/monitoring/dashboards/atuin-dashboard-configmap.yaml

# Revert to local-only mode on each machine:
chezmoi edit-config
# Set: atuin_sync_address = ""
chezmoi apply
# Atuin continues working locally without sync
```
